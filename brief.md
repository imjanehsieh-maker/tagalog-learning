# 他加祿語學習 PWA Brief

更新：2026-04-30（從「整合兩網頁」改成「擴展為日常 + 教材兩用 PWA」）

---

## 真實使用情境

小豆學他加祿語為了傳道、關心菲律賓弟兄姊妹。除了系統性教材學習，更多時候是日常情境累積：對話現場聽到新詞、被對方教念、傳道前要快速複習某人之前教過的詞。

| 場景 | 在哪 | 要做什麼 | 痛點 |
|---|---|---|---|
| A. 對話當下 | 聚會、聊天現場 | 想說一句話卻不會→快查；對方教我一個詞→快記 | 要快、聽到音不會拼、想錄音 |
| B. 路上/睡前 | 捷運、走路、躺床 | 整理今天學到的、零碎時間複習 | 單手用、不正襟危坐 |
| C. 教材學習 | 桌前 | 跟教材走 W1–W15 | 已有現成網頁 |
| D. 傳道前準備 | 出門前 | 複習等下要見的朋友之前教過的詞 | 按「人」「主題」搜尋 |

**核心設計原則**：一個入口（PWA icon）吃下所有場景，**零切換**。

---

## 專案目標

擴展現有他加祿語網頁，做成 PWA（手機加到主畫面像 app），整合：
1. 既有教材（W1–W10）+ 新教材（W11–W15）
2. 日常快速記錄 + 我的詞庫（含錄音）
3. Firebase 後端（資料雲端同步，換手機不掉）

## PWA 功能規格

### 1. 教材模式（既有功能 + 擴充至 W15）
- 📋 單字瀏覽（按週次/分類/搜尋/收藏）
- 🎴 閃卡模式
- 🎯 隨機測驗
- 📖 課程內容（對話、文法、例句）

### 2. 快速記錄（新）
- 一鍵新增詞：拼字（不確定可寫音）、中文意思、可附**錄音**
- 標籤：誰教的（朋友名）、主題（傳道/分租/食物/...）、學會程度
- 收件匣模式：當下不整理，事後再分類

### 3. 我的詞庫（新）
- 看自己累積的詞（跟教材詞庫互通，不重複）
- 按「誰教的」「主題」「學會程度」篩選與搜尋
- 標記「實戰確認」（教材學的詞，現場用過了）

### 4. 後端與部署
- **Firebase Firestore**（資料同步）+ Firebase Auth（單一用戶就好）
- **PWA**：manifest + service worker，iPhone 加到主畫面
- 錄音用 Web Audio API，音檔放 Firebase Storage

### 5. 場景對應一覽

| 場景 | PWA 操作 |
|---|---|
| A. 對話當下 | 主畫面 icon → 快速記 → 打字+錄音，3 秒搞定 |
| B. 路上複習 | 同 icon → 我的詞庫 → 看今天記的 |
| C. 教材 | 同 icon → 課程 → W12 |
| D. 傳道前 | 同 icon → 搜尋朋友名或標籤 |

## 派工分配

### 陪讀（教材內容整理）
從 W11–W15 PDF 提取結構化內容，輸出 JSON，給工程師接進網頁。
- 單字：詞、詞性、中文意思、例句、發音提示
- 對話：情境、角色 A/B 對話文、中文翻譯
- 文法重點
- 格式：跟既有網頁的資料結構一致（工程師會先給 schema）

### 工程師（PWA 開發 + 整合）
1. 擴展現有 `index.html` + `tagalog.html` → 合併為單一 PWA
2. 新增「快速記錄」「我的詞庫」兩個分頁
3. Firebase Firestore + Storage（錄音檔）
4. PWA manifest + service worker（iPhone 加到主畫面）
5. 接陪讀產出的 W11–W15 JSON 進去

**並行**：工程師先做 PWA 架構 + 快速記錄/我的詞庫（不依賴教材），陪讀整理教材，最後合併。

### 過渡期
PWA 完成前，小豆用 iPhone 內建備忘錄一個檔案隨手記，網頁完成後一次匯入。
